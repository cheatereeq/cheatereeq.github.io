What is Responsive Web Design?
Nowadays, we use the Internet not only on desktop computers but above all on mobile devices such as smartphones, smartwatches or laptops. The demand for websites adapted to mobile devices is constantly growing. Mobile phones have a completely different screen resolution, that's why websites created for computers look bad on smartphones. Writing several pages for different screens is too time consuming, so a new way was created that allowed defining a different page appearance for other devices in one project.
What tags should be used in RWD?
There are several tags that you should be able to use to write a good responsive website:
  • @media
  • vh
  • vw
  • initial-scale
Media tag
The @media tag is most important when creating a responsive website. Used to define the appearance of the page for different screen resolutions. It is used in external styles. To make the site work well on different devices, use it several times each time designing the site for different resolutions.
Correctly written code should look like this:
@media (min-width: 768px){
   .title{
      font-size: 6vw;
   }

   .subtitle{
      font-size: 3vw;
   }
}
Size of website
Typically, pages are written for desktops, laptops, tablets and smartphones. The following resolutions are the borders for different versions of the page:
  • 576px - smartphone
  • 768px - tablets
  • 992px - laptops
  • 1200px - desktop PC
Summary
Nowadays RWD is developing dynamically. Most of the sites known to us can be viewed on the above devices. Depending on the size of the device, the various elements on the page take on a different look. It often happens that some elements are too burdensome for smartphones, so the developers remove them from the phone version. To sum up - knowledge of RWD is necessary to create a good website regardless of the screen on which it will be displayed.
